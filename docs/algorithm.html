2. Data Interface

2. 数据接口

The current dataset is an Excel table with one row per experiment or candidate alloy. Training requires both features and labels. Inference can accept the same feature columns without labels; if labels are present, they are dropped to avoid leakage.

当前数据是 Excel 表格，每一行代表一个实验或候选合金。训练需要特征和标签。 推理只需要特征列；如果输入文件包含标签列，程序会自动删除标签列，避免信息泄漏。

Required Columns

必需列

Group 分组 Columns
Composition 成分 Al, Co, Cr, Fe, Ni, Ti, Mo, Nb
Categorical / encoded process 类别或编码工艺 Eutectic, Preparation, Processing, Tensile/compress
Continuous process 连续工艺参数 Cold CR_%, Annealing_Temp_K, Annealing_time_h, Aging_Temp_K, Aging_Time_h
Test condition 测试条件 Rate_S-1, Tes_Temp_K
Targets for training only 仅训练需要的标签 Strength_MPa, Plasticity_%

Composition values are normalized internally by row sum. Categorical columns must already be integer encoded from 0 to the maximum category id.

成分列会在程序内部按行归一化。类别列需要提前编码成从 0 开始的整数。

3. Algorithm Overview

3. 算法概览

The model is ATON with a residual output adapter. The neural ATON backbone remains the main algorithm. The adapter is attached at the output level and learns the residual error left by ATON.

当前模型是带输出残差适配器的 ATON。神经网络 ATON 主干仍是核心算法； 残差适配器接在输出端，用来学习 ATON 未解释的误差。

ATON Backbone

ATON 主干

  • Composition branch: encodes 8 normalized alloy composition features.
  • 成分分支：编码 8 个归一化合金成分特征。
  • Processing branch: embeds categorical processing variables and encodes continuous processing variables.
  • 工艺分支：类别工艺变量做 embedding，连续工艺变量进入 MLP。
  • Trajectory state branch: encodes one scalar t.
  • 轨迹状态分支：只编码一个标量 t。
  • Test branch: encodes strain rate and test temperature.
  • 测试条件分支：编码应变速率和测试温度。
  • Fusion: processing and state information gate the composition representation; all encoded branches are then fused for two-task prediction.
  • 融合：工艺和状态信息调制成分表征，随后多分支特征融合并预测两个目标。

Trajectory State Encoding

轨迹状态编码

The trajectory state is a single scalar t. During supervised training, rows with the same composition are sorted by sigma_i * epsilon_i, i.e. Strength_MPa * Plasticity_%. The resulting rank is used as t. Different compositions are ranked independently.

轨迹状态只有一个标量 t。监督训练时，同一成分内部按 sigma_i * epsilon_i，也就是 Strength_MPa * Plasticity_% 排序， 排名作为 t。不同成分组合之间互不干扰，分别排序。

During validation and inference, labels are not used. The model starts from an initial row-order ranking, predicts strength and plasticity, recomputes predicted_strength * predicted_plasticity, updates t, and repeats until convergence or the maximum iteration count is reached.

验证和推理阶段不能使用标签。模型先按初始行顺序构建 t，预测强度和塑性， 再用 预测强度 * 预测塑性 重新排序并更新 t，不断迭代直到收敛或达到最大迭代次数。

Residual Adapter

残差适配器

After ATON is trained, a residual adapter is fitted on the training fold. It does not replace ATON. Its target is:

ATON 训练完成后，在训练折上拟合残差适配器。它不会替代 ATON。它的学习目标是： 

residual_target = y_true - alpha * y_ATOM
ATOM_prediction = alpha * y_ATOM + residual_adapter(features)

This gives the final model better small-data tabular performance while preserving the ATON architecture as the main prediction framework.

这样既保留 ATON 作为主预测框架，又能提升小样本表格数据上的泛化性能。

4. Training Workflow

4. 训练流程

Run the full training workflow from the project root:

在项目根目录运行完整训练流程：

python trainer.py

What Happens Internally

内部执行步骤

  1. Load data.xlsx and validate all required columns.
  2. 读取 data.xlsx 并检查必需列。
  3. Split the data with KFold(n_splits=5, shuffle=True, random_state=42).
  4. 使用 KFold(n_splits=5, shuffle=True, random_state=42) 进行五折划分。
  5. For each fold, build training trajectory states from labels only inside the training fold.
  6. 每折只在训练折内部使用标签构建轨迹状态。
  7. Train ATON and select the best epoch using label-free self-consistent validation loss.
  8. 训练 ATON，并用无标签自洽验证损失选择最佳 epoch。
  9. Fit the residual adapter on the training fold residuals.
  10. 在训练折残差上拟合残差适配器。
  11. Predict the held-out fold with self-consistent inference.
  12. 对 held-out 折做自洽预测。
  13. Concatenate all five held-out predictions and calculate final metrics once on the full OOF vector.
  14. 拼接五折 held-out 预测，最后只在完整 OOF 预测上统一计算一次指标。

Useful Arguments

常用参数

python trainer.py \
  --data data.xlsx \
  --output-dir checkpoints \
  --epochs 300 \
  --batch-size 32 \
  --lr 1e-3 \
  --patience 50 \
  --sc-max-iters 30 \
  --sc-tol 1e-3
Argument Meaning 含义
--data Excel or CSV data path. Excel 或 CSV 数据路径。
--output-dir Directory for checkpoints, OOF predictions, and summary JSON. 保存 checkpoint、OOF 预测和 JSON 汇总的目录。
--epochs Maximum training epochs per fold. 每折最大训练轮数。
--patience Early stopping patience. 早停耐心轮数。
--sc-max-iters Maximum self-consistent trajectory iterations. 自洽轨迹迭代最大次数。
--sc-tol Convergence tolerance for self-consistent prediction. 自洽预测收敛阈值。

5. Inference Workflow

5. 推理流程

Inference is label-free. If the input file contains Strength_MPa or Plasticity_%, the script removes them before trajectory construction.

推理过程无标签。如果输入文件包含 Strength_MPa 或 Plasticity_%， 脚本会先删除这些列，再构建轨迹状态。 

python inference.py candidates.xlsx predictions.xlsx \
  --checkpoint checkpoints/model_best.pth \
  --max-iters 30 \
  --sc-tol 1e-3

Prediction Columns

预测输出列

Residual correction is an internal part of the ATOM model. The user-facing prediction file exposes only one final prediction per target.

残差校正是 ATOM 模型内部机制。面向用户的预测文件中，每个目标只输出一个最终预测结果。

Column Meaning 含义
ATOM_Predicted_Strength_MPa, ATOM_Predicted_Plasticity_% Final ATOM prediction after self-consistent trajectory inference and internal residual correction. 经过自洽轨迹推理和内部残差校正后的最终 ATOM 预测结果。

6. Replacing the Dataset

6. 替换成自己的数据

  1. Prepare an Excel or CSV file with the required columns listed in Section 2.
  2. 准备一个 Excel 或 CSV 文件，包含第 2 节列出的所有必需列。
  3. Keep target columns only for training data. Candidate files for inference can omit them.
  4. 训练数据保留标签列；推理候选数据可以不包含标签列。
  5. Encode categorical columns as integers. Example: Preparation = 0, 1, 2, ....
  6. 将类别列编码为整数。例如 Preparation = 0, 1, 2, ...。
  7. Run python trainer.py --data your_data.xlsx.
  8. 运行 python trainer.py --data your_data.xlsx。
  9. Use the generated checkpoints/model_best.pth for inference.
  10. 使用生成的 checkpoints/model_best.pth 做推理。

If your new dataset has different element columns or different process columns, edit the column lists in src/data.py first. The rest of the pipeline reads those constants.

如果你的新数据元素列或工艺列不同，先修改 src/data.py 中的列名常量。 训练、推理和搜索空间生成都会读取这些常量。

7. Search Space Generation

7. 搜索空间生成

build_search_space.py creates virtual candidates by copying one reference row and scanning Cold CR_%, Annealing_Temp_K, and Annealing_time_h. The strain rate is fixed to 1e-3.

build_search_space.py 会复制一个参考样本，并扫描 Cold CR_%、Annealing_Temp_K 和 Annealing_time_h。 应变速率固定为 1e-3。 

python build_search_space.py --data data.xlsx --output search_space.csv
python inference.py search_space.csv search_predictions.xlsx --checkpoint checkpoints/model_best.pth

9. Common Issues

9. 常见问题

Problem 问题 Fix 解决方法
Missing required columns 缺少必需列 Check your table against Section 2 or update src/data.py. 按第 2 节检查表格，或修改 src/data.py。
Category embedding index error 类别 embedding 索引错误 Ensure categorical columns are non-negative integer ids and training data covers the ids used at inference. 确保类别列是非负整数，并且推理中的类别编号训练集出现过。
Inference seems to use labels 怀疑推理使用了标签 It does not. inference.py explicitly drops target columns if present. 不会。inference.py 会显式删除目标列。
Notebook cannot find OOF predictions Notebook 找不到 OOF 预测 Run python trainer.py first, then rerun the notebook. 先运行 python trainer.py，再运行 notebook。
